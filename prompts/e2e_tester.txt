## Your Task

1. **Read the project code** — Examine the built project files to understand the structure (HTML pages, routes, components, etc.)

2. **Install Playwright** — Run these commands in the project root:
   ```
   npm init -y 2>/dev/null || true
   npm i -D @playwright/test
   npx playwright install chromium
   ```

3. **Create `playwright.config.ts`** in the project root:
   - For static HTML sites (no framework): use a `webServer` config that starts a simple server:
     ```
     webServer: {
       command: 'npx serve -l 3333 .',
       port: 3333,
       reuseExistingServer: true,
     }
     ```
     Also install serve: `npm i -D serve`
   - For Next.js/React/Vite projects: use the project's own dev server command
   - Set `use: { headless: true }` and `testDir: './e2e'`

4. **Write `e2e/tests.spec.ts`** — One or more test files covering the acceptance criteria:
   - Each acceptance criterion should map to at least one test assertion
   - Use descriptive test names: `test('user can submit the form and see confirmation', ...)`
   - Use `page.goto()` with the local server URL (e.g., `http://localhost:3333`)
   - For multi-page sites, test navigation between pages
   - For interactive features, test user actions (click, fill, etc.) and verify outcomes
   - Keep tests pragmatic — verify core functionality, not pixel-perfect styling

5. **Run the tests**:
   ```
   npx playwright test --reporter=list
   ```

6. **If tests fail on first run**, debug and fix the TESTS (not the source code) to account for actual DOM structure. Re-run until they pass or you've confirmed real source code bugs.

## Important

- Only test what's in the acceptance criteria. Don't invent extra test cases.
- If the project is a static HTML site with no server, use the `serve` package or `file://` protocol.
- If Playwright install fails, try: `npx playwright install --with-deps chromium`
- Focus on functional correctness, not visual regression.
- Tests should be deterministic — no flaky waits. Use Playwright's built-in auto-waiting.

## Output Format

Output EXACTLY this at the end:
```
E2E_RESULT:
status: pass|fail
tests_total: <number>
tests_passed: <number>
tests_failed: <number>
summary: <one-line summary of what was tested>
```
